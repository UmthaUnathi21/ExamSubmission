Critical Reflection

UI and UX

Umtha-Unathi Mavuso August, 2023 500

Data visualization with regards to my Implementation of UI and UX has been centered around ease of access to all infromation as stated in the brief that the user should be able to access any part of the given pages at any point in time along their exploration of the website. Hence tags are placed at each juncture about 1/4 length of the page apart so users are always able to navigate to different sections of the website.

In my experience it is essential to keep information to a minimum to avoid information overloadding.I have organised the main and most important blog sections at the top of the navbar as well as made the navbar accessible from any point on the page. I have also made the page interactive by changing colour of the text when the mouse hovers over the text indicating to the user that the text is a button and clickable. They organisation of the page has been directed towards incorporating the use of negative space to instill a more free-flowing feel to the website as a whole.

My weak points in the creation of these projects has been my ability to apply the NASA API's and having simple and readable pages. My emphasis this time around thus has been on stripping down the elements that distracted from the effiecient understanding/comprehension of information on the page and has been about implementing effect UI/UX in conjunction with the appliction of the API format.

This method of design is more along the lines of retro web design. Envoking feelings of nostalgia like using the earliest MacBook models. I will expand on this by adding retro icons and more stylized UI containers.

I think I have improved my understanding of data visualisation and interactivity of data elements. This is seen in the implemented interactive static scatterplot, heatmap and world population graphs. I still struggle somewhat with bargraph synthesis and aim to improve that aspect as I continue to learn.
